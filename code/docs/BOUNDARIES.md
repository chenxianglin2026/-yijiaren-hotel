# 项目边界定义 (Boundaries)

> 目的：明确每个项目的"可改"与"不可改"区域，防止误操作。

---

## 一、伊家人酒店 (yijiaren)

### 🔴 不可改区域 (Tier 3 - 需人工审批)
| 边界 | 原因 | 位置 |
|------|------|------|
| 生产容器 yijiaren-app | 在线服务，停机=业务中断 | VPS Docker |
| 生产数据库 /data/yijiaren-db | 客户数据，不可丢 | VPS 宿主机 |
| 微信小程序已发布版本 | 线上用户在用 | 微信后台 |
| iptables 规则 | 影响所有端口安全 | VPS |
| 宿主机 nginx (443) | HTTPS 入口 | VPS /root/scenic-nginx.conf |

### 🟡 谨慎修改 (Tier 2)
| 边界 | 说明 |
|------|------|
| 容器内 nginx config | 需 reload，不影响运行 |
| 后端 API 响应格式 | 影响小程序/管理后台 |
| 数据库 Schema | 需迁移脚本 |
| GitHub 仓库 main 分支 | force push 禁止 |

### 🟢 自由修改 (Tier 0-1)
| 边界 | 说明 |
|------|------|
| 管理后台 HTML/JS | 前端代码 |
| 小程序 JS/WXML/WXSS | 前端代码 |
| 后端业务逻辑 (.py) | 可测试 |
| 测试文件 | 新增/修改 |
| 文档 | 任意修改 |
| 种子数据 seed_mock.py | 开发数据 |

### 📡 外部依赖 (Tier 3 - 无法控制)
| 依赖 | 状态 |
|------|------|
| 微信小程序审核 | 等待中 |
| 微信支付商户号 | 待申请 |
| TTLock 门锁 API | 待审核 |
| 携程/美团/飞猪 OTA | 框架完成，缺密钥 |
| 充电桩 washpayer.com | 未对接 |

## 二、KeepSafe 防丢器 (keepsafe)

|------|------|

| 后端业务逻辑 | 可测试 |

## 三、景区智慧系统 (scenic)

### 🔴 不可改区域
| 边界 | 原因 |
|------|------|
| 生产容器 scenic-backend | 在线服务 |
| 宿主机 nginx 443 /scenic/ | HTTPS 入口 |

### 🟡 谨慎修改
| 边界 | 说明 |
|------|------|
| OTA 框架 | 810行，影响携程/美团/飞猪对接 |
| nginx /scenic/ 代理 | 影响景区 API |

### 🟢 自由修改
| 边界 | 说明 |
|------|------|
| 管理后台 | 前端代码 |
| 后端业务逻辑 | 可测试 |
| 小程序代码 | 前端（AppID 到手前无线上版本） |

## 四、全局红线

- ❌ 不碰生产数据库 (SELECT 可，DROP/ALTER 不可)
- ❌ 不 force push GitHub main 分支
- ❌ 不修改 Docker 容器的 nginx 配置（走宿主机 443）
- ❌ 不修改 iptables 规则
- ❌ 不修改微信已发布版本
- ⚠️ 任何 Tier 3 操作需陈总确认
